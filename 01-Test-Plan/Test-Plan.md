# 🧪 Test Plan – OWASP Juice Shop

## 📌 Introduction

This Test Plan defines the strategy, scope, approach, and deliverables for testing the OWASP Juice Shop application.

The objective is to ensure functional correctness, identify defects, and evaluate security weaknesses.

---

## 🎯 Objectives

- Validate core application functionalities
- Identify functional and non-functional defects
- Perform negative testing scenarios
- Detect basic security vulnerabilities
- Ensure proper user flow across the application

---

## 📦 Scope

### ✅ In Scope

- Authentication (Login & Registration)
- Product Catalog (Main Page)
- Search Functionality
- Shopping Cart
- Checkout Process

---

### ❌ Out of Scope

- Performance Testing
- Load Testing
- Automation Testing
- Backend/API deep testing

---

## 🧪 Testing Types

The following testing types will be applied:

- Functional Testing  
- Negative Testing  
- Exploratory Testing  
- Usability Testing (basic)  
- Security Testing (focused on common web vulnerabilities)

---

## 🧭 Test Approach

Testing will be performed manually using exploratory and structured test cases.

The approach includes:

- Test case execution based on requirements
- Validation of positive and negative scenarios
- Security-focused input testing
- End-to-end user flow validation

---

## 🛠️ Test Environment

- Application: OWASP Juice Shop
- Browser: Latest version of Chrome
- Operating System: Windows 10 / Windows 11
- Network: Stable internet connection

---

## 📋 Test Deliverables

- Test Plan
- Test Cases
- Smoke Test Suite
- Regression Test Suite
- Test Execution Reports
- Bug Reports

---

## 🚪 Entry Criteria

- Application is accessible
- Test environment is stable
- Requirements are defined

---

## 🏁 Exit Criteria

- Critical test cases executed
- Major defects documented
- Core functionalities validated
- Smoke tests passed

---

## ⚠️ Risks

- Application contains intentional vulnerabilities
- Unexpected behavior due to insecure design
- UI inconsistencies across browsers

---

## 📊 Success Criteria

- All critical modules tested
- Major functional flows validated
- Security issues identified and documented
- Test coverage achieved for defined scope
